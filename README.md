# SkillForge UML Design

### 1- Use Case Diagram  
Users can log in or register to access the platform, students can enroll in courses after browsing available options and update their progress as they complete lessons and quizzes Instructors can create and manage courses by uploading lessons and designing quizzes Admins manage users and approve or remove courses, and they can view analytics

### 2- Activity Diagram  
This activity diagram illustrates the process of enrolling in a course on the SkillForge platform.
It shows how a student logs in, selects a course, and proceeds through decisions that check login validity and course availability before enrollment and database update.

### 3- Class Diagram  
The class diagram represents SkillForge's system structure through classes, inheritance hierarchies, 
and relationships with precise multiplicities. 
It shows how users interact with courses, lessons, and quizzes, 
while service classes handle business operations and data files operations.

### 4- Sequence Diagram  
This sequence diagram shows how the Admin, Instructor, and Student interact with the SkillForge system over time. It demonstrates the step-by-step flow of login, course management, lesson completion, assessment, and certificate generation as each user role performs its tasks sequentially.
